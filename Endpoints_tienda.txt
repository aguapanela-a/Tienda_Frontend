post -> Endpoint para añadir un monto (Puede ser abono o deuda, ahi mando el tipo)

Este endpoint maneja varias reglas de negocio

1) Desde el frontend, siempre se ingresaran, valores positivos, ya sea deuda o abono se maneja positivo. Y cuando se muestren en pantalla, también serán positivos.

2) El backend, si recibe una deuda lo vuelve positivo, si recibe un abono lo vuelve negativo. Esto pues requiere el calculo, sin embargo cuando retorna el valor del abono, lo hace negativo, el frontend debe siempre recibirlo y manejarlo negativo. El frontend no realizara ningún tipo de calculo, pues toda la lógica la maneja el backend. 

3) A un cliente tipo invitado, no se le pueden hacer abonos, solo deudas. 

	Ej: POST http://localhost:8087/clientes/montos

	Envio desde el frontend: 
{
  "id_cliente": 1,
  "descripcion": "Compra bebida",
  "valor": 500,
  "tipo_monto": "abono",
  "fecha": "01-04-2026"
}

	Backend Devolver:

{
    "id_monto": 2,
    "id_cliente": 1,
    "descripcion": "Compra bebida",
    "valor": -500,
    "tipo_monto": "abono",
    "fecha": "01-04-2026"
}


 	Ej de envió de abono
Request:
{
  "id_cliente": 1,
  "descripcion": "Compra bebida",
  "valor": 4000,
  "tipo_monto": "deuda",
  "fecha": "01-04-2026"
}

Response:
{
    "id_monto": 4,
    "id_cliente": 1,
    "descripcion": "Compra bebida",
    "valor": 4000,
    "tipo_monto": "deuda",
    "fecha": "01-04-2026"
}
